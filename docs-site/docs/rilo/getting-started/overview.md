Rilo runs a checkpointed media pipeline:

1. script
2. voiceover
3. keyframes
4. segments
5. compose
6. align subtitles (optional)
7. burn-in subtitles (optional)

Project state is stored under `projects/<project>/` so runs can resume and regenerate downstream stages safely.
